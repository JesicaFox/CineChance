# Обновление медиа-типов (Media Type Migration)

## Задача
Обновить существующие записи в таблице `watchList` с корректируемыми медиа-типами:
- `movie` → `movie` (без изменений)
- `tv` → `tv` (без изменений)  
- `movie/tv` → `anime` (если обнаружено аниме по жанру и языку)
- `movie/tv` → `cartoon` (если это мультфильм)

**Важно:** Это также обновляет связанные записи в `RatingHistory` и `RewatchLog` - это необходимо из-за foreign key constraints в БД.

## Способ 1: Web Interface (рекомендуется) ✅

### Самый простой способ - используйте встроенную админ-панель:

1. Запустите приложение:
   ```bash
   npm run dev
   ```

2. Откройте админ-страницу миграции:
   ```
   http://localhost:3000/admin/migrate
   ```

3. Нажмите кнопку "▶️ Run Migration"

4. Дождитесь завершения - будет показана статистика

**Это не требует:**
- Никаких скриптов ts-node
- Переменных окружения  
- Команд в терминале
- Никаких ошибок с подключением

## Способ 2: API (для автоматизации)

Если нужно запустить миграцию программно:

```bash
curl -X POST http://localhost:3000/api/admin/migrate-media-types \
  -H "Content-Type: application/json"
```

С опциональным токеном (если установлен ADMIN_MIGRATION_TOKEN):
```bash
curl -X POST http://localhost:3000/api/admin/migrate-media-types \
  -H "x-admin-token: your-secret-token" \
  -H "Content-Type: application/json"
```

## Что происходит при миграции

1. **Загружаются все записи** из таблицы `watchList`
2. **Для каждой записи:**
   - Отправляется запрос к TMDB API
   - Получаются жанры и язык оригинала
   - Определяется тип: anime, cartoon, movie или tv
   - Если тип изменился:
     - **Сначала обновляются** связанные записи в `RatingHistory` и `RewatchLog` (это важно!)
     - **Затем обновляется** сама запись в `watchList`
3. **После завершения:**
   - Выводится статистика (всего, обновлено, пропущено, ошибок)
   - Любые ошибки логируются

## Требования

- ✅ `npm run dev` запущен
- ✅ DATABASE_URL сконфигурирована (.env.local)
- ✅ TMDB_API_KEY сконфигурирована (для определения типов)

## Результат

После миграции:
- Фильтры по типам контента будут работать корректно
- Аниме и мультфильмы будут отображаться с правильным типом
- Все остальные данные (рейтинги, даты, теги) остаются без изменений
- Все связанные записи в `RatingHistory` и `RewatchLog` будут синхронизированы

## Технические детали

### Проблема с Foreign Keys
Таблицы `RatingHistory` и `RewatchLog` ссылаются на `watchList` через composite key `(userId, tmdbId, mediaType)` с ограничением `onUpdate: NoAction`. 

Это значит:
- ❌ Нельзя просто обновить `mediaType` в `watchList`
- ✅ Нужно сначала обновить `mediaType` в `RatingHistory` и `RewatchLog`
- ✅ Только потом обновлять в `watchList`

Код миграции это учитывает и выполняет обновления в правильном порядке.
