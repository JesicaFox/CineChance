# Почему скрипт падает с ошибкой DATABASE_URL

## Проблема: Next.js загружает .env автоматически, а ts-node - нет

### Как работает в приложении:
```
┌─────────────────────────────────────┐
│  .env.local (не закоммичен в git)   │
│  ↓                                  │
│  Next.js автоматически загружает    │
│  ↓                                  │
│  process.env.DATABASE_URL доступна  │
│  ↓                                  │
│  Prisma использует переменную        │
└─────────────────────────────────────┘
```

### Как работает в скрипте (было):
```
┌─────────────────────────────────────┐
│  .env.local (не закоммичен в git)   │
│  ↓ (ts-node не знает о ней!)        │
│  ts-node запускается БЕЗ .env.local  │
│  ↓                                  │
│  process.env.DATABASE_URL не найдена│
│  ↓                                  │
│  Prisma падает с ошибкой            │
└─────────────────────────────────────┘
```

## Решение

Скрипт теперь использует `import 'dotenv/config'` в начале для явной загрузки .env файлов.

### Способ 1: Используйте существующий .env.local (рекомендуется)

Ваш .env.local файл уже содержит DATABASE_URL (используется при локальной разработке). Скрипт теперь его загружает:

```bash
npx ts-node scripts/migrate-anime-cartoon-mediaType.ts
```

Скрипт автоматически загрузит .env.local из корня проекта.

### Способ 2: Передайте переменную при запуске

```bash
DATABASE_URL="postgresql://..." npx ts-node scripts/migrate-anime-cartoon-mediaType.ts
```

### Способ 3: Экспортируйте переменную в текущей сессии

```bash
export DATABASE_URL="postgresql://..."
npx ts-node scripts/migrate-anime-cartoon-mediaType.ts
```

## Где найти DATABASE_URL?

Ваш DATABASE_URL уже настроен, он находится:

1. **На локальной машине**: в файле `.env.local` (не закоммичен в git)
2. **На Vercel**: в Settings → Environment Variables (если развёрнуто там)
3. **В CI/GitHub Actions**: в секретах репозитория (если есть)

Если используете локальную разработку, просто запустите:
```bash
npm run dev
```

Это подтвердит, что DATABASE_URL правильно настроена, и скрипт будет работать.

## Теперь скрипт работает как Next.js приложение

✅ Загружает .env.local (как Next.js)
✅ Загружает .env (как Next.js)
✅ Использует переменные окружения платформы (как Vercel)
✅ Не требует дополнительной конфигурации
