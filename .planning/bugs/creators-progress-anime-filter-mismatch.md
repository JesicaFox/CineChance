# Bug: Неверный расчет процента заполнения фильмографии режиссеров
Дата: 2025-03-12
Slug: creators-progress-anime-filter-mismatch
Status: **FIXED**

## Описание пользователя
На странице /profile/creators для режиссеров, у которых в фильмографии присутствуют только аниме и/или мультфильмы, отображается некорректное значение процента заполнения, например "3/0" (3 просмотренных из 0 общих). Это происходит потому что общее количество фильмов (total_movies) фильтруется для исключения аниме и мультфильмов, а количество просмотренных (watched_movies) — нет.

## Шаги воспроизведения
1. Убедиться, что у пользователя в watchlist есть фильмы/сериалы, которые являются аниме (жанр Animation + язык ja) или мультфильмами (жанр Animation + не ja), и которые сняты определенным режиссером.
2. При этом у этого режиссера в полной фильмографии (TMDB) НЕТ кинофильмов или TV-шоу, которые не являются аниме/мультфильмами.
3. Перейти на страницу /profile/creators
4. Найти данного режиссера в списке
5. Увидеть: watched_movies > 0, total_movies = 0, progress_percent = 0% (или некорректный расчет)

## Ожидаемое vs Фактическое
- **Ожидалось:** total_movies и watched_movies считаются на одном и том же наборе фильмов (только не-аниме/мультфильмы). Если у режиссера только аниме/мультфильмы, то total_movies=0, watched_movies=0, progress=0%.
- **Происходило:** watched_movies включает аниме/мультфильмы, а total_movies — нет. Результат: 3/0, 5/0 и т.д.

## Root Cause
Файл: `src/app/api/user/achiev_creators/route.ts`
Функция/компонент: `GET` (route handler), режим `singleLoad=true`

**Точное объяснение:**

В блоке `if (singleLoad)` для каждого создателя значения `watched_movies`, `rewatched_movies`, `dropped_movies` брались из `baseCreatorsData`, которые формировались из watchlist пользователя БЕЗ фильтрации аниме/мультфильмов. В то время как `total_movies` вычислялся ПОСЛЕ фильтрации фильмографии (исключая аниме/мультфильмы).

## Fix Applied
**Файл:** `src/app/api/user/achiev_creators/route.ts`
**Строки:** ~706-751

Исправление:
1. Созданы Set'ы из watchlist данных: `watchedMovieIds`, `rewatchedMovieIds`, `droppedMovieIds`
2. Пересечение с `uniqueMovieIds` (отфильтрованная фильмография без аниме/мультфильмов)
3. Теперь `watched_movies`, `rewatched_movies`, `dropped_movies` вычисляются на том же наборе, что и `total_movies`

## Acceptance критерии исправления
- [x] В режиме `singleLoad=true` все показатели (`watched_movies`, `rewatched_movies`, `dropped_movies`, `total_movies`) вычисляются на основе ОДНОГО отфильтрованного списка фильмов (исключая аниме/мультфильмы).
- [x] После исправления для режиссера с только аниме/мультфильмами: `total_movies=0`, `watched_movies=0`, `progress_percent=0`.
- [x] Для остальных режиссеров процент заполнения корректен.
- [x] Регрессий в существующих тестах нет.
