# Bug: My Movies - Ограничение в 100 фильмов на вкладку
Дата: 2026-03-10

## Описание
На странице "Мои фильмы" (/my-movies) при загрузке без фильтров отображается только 100 фильмов на каждой вкладке (Просмотрено, Хочу посмотреть, Брошено). При прокрутке вниз подгружаются ещё фильмы, но общий лимит составляет 100 записей, после чего загрузка прекращается (hasMore становится false). Это происходит даже если у пользователя в коллекции более 100 фильмов.

Ожидается: При отсутствии фильтров система должна корректно реализовать бесконечную пагинацию, загружая все фильмы пользователя по 20 штук на страницу, без искусственных лимитов.

## Шаги воспроизведения
1. Войти в систему как пользователь с более чем 100 фильмами в любом статусе
2. Перейти на страницу /my-movies
3. Выбрать вкладку с большим количеством фильмов (>100)
4. Прокрутить вниз до загрузки всех фильмов
5. Наблюдать: после загрузки ~100 фильмов подгрузка прекращается

## Ожидаемое поведение
- При отсутствии TMDB-фильтров используется классическая пагинация с skip/take
- Должны загружаться все фильмы пользователя, по 20 на страницу
- hasMore должен правильно отражать наличие дальнейших страниц

## Фактическое поведение
- Используется стратегия "fetch-all-with-buffer" для всех случаев
- BUFFER_SIZE = 100 при отсутствии TMDB-фильтров
- Нет использования skip, всегда берутся первые N записей из БД
- Это ограничивает общее количество загружаемых фильмов 100

## Локализация
- **Файлы:** `src/app/api/my-movies/route.ts`
- **Функция:** GET endpoint, секция regular tabs (строки 309-519)
- **Root cause:** Использование неправильной стратегии пагинации для случая без TMDB-фильтров

## Root Cause
Код использует:
```typescript
watchListRecords = await prisma.watchList.findMany({
  take: BUFFER_SIZE, // 100 или 1000
  // skip отсутствует
});
```
В то время как для hidden tab уже реализовано правильное ветвление:
```typescript
if (hasTMDBFilters) {
  // fetch all then filter
} else {
  // DB-level пагинация: skip/take
}
```

Требуется такое же ветвление для regular tabs.

## Acceptance Criteria
- [ ] При отсутствии TMDB-фильтров используется DB-level пагинация (skip/take)
- [ ] При активных TMDB-фильтрах используется fetch-all-then-filter с BUFFER_SIZE=5000
- [ ] Первая страница без фильтров возвращает ровно 20 фильмов
- [ ] hasMore корректно отражает наличие следующих страниц
- [ ] totalCount возвращает полное количество для skip/take, отфильтрованное для buffer
- [ ] Никаких дубликатов между страницами
- [ ] Существующие тесты проходят

## RED Test Spec
**Тест:** Загрузка 150+ фильмов без фильтров должна вернуть все записи через 8 страниц по 20 штук.
**Ожидаемый результат текущего кода:** Тест падает - максимум 100 фильмов, hasMore=false преждевременно.

## GREEN Fix
Добавить ветвление в `route.ts` regular tabs:
- `hasTMDBBasedFilters` → buffer strategy (5000)
- `!hasTMDBBasedFilters` → skip/take strategy
- Унифицировать hasMore и totalCount calculation

## REFACTOR Plan
1. Создать `src/lib/watchlist-pagination.ts` с двумя стратегиями
2. Использовать фабрику `getPaginationStrategy(hasTMDBBasedFilters)`
3. Убрать дублирование между hidden и regular tabs
4. Вынести константы BUFFER_SIZE в конфиг
