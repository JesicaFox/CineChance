# Bug: Recommendations page: Anime and Cartoon filters return "No available recommendations"

Дата: 2026-03-25

## Описание

При выборе на странице /recommendations фильтров типов контента "Аниме" или "Мульт" (по отдельности или вместе) система возвращает сообщение "Нет доступных рекомендаций по выбранным фильтрам", даже если у пользователя в библиотеке есть соответствующие элементы. Фильтры "Фильм" и "Сериал" работают корректно.

## Похожие прошлые баги

- `.planning/debug/resolved/cartoon-filter-no-recommendations.md` (2026-02-19) — ранее исправлена проблема, когда фильтр "Мульт" не был включен в логику фильтрации API. На текущий момент фильтр присутствует, но логика определения типа контента не работает.

## Шаги воспроизведения

1. Убедиться, что в пользовательском списке (My Movies) есть как минимум несколько аниме (японские анимационные работы) и/или мультфильмов (не-японские анимационные работы).
2. Перейти на страницу /recommendations.
3. В форме фильтров выбрать тип "Аниме" (и/или "Мульт"), оставив включенными только эти типы.
4. Нажать кнопку "Подобрать".
5. Увидеть сообщение "Нет доступных рекомендаций по выбранным фильтрам".

## Ожидаемое поведение

При выборе фильтров "Аниме" и/или "Мульт" система должна находить в библиотеке пользователя соответствующие элементы (с жанром анимации id=16) и возвращать случайную рекомендацию, учитывая остальные фильтры (списки, рейтинг, жанры, теги).

## Фактическое поведение

Система возвращает ошибку "Нет доступных рекомендаций по выбранным фильтрам", так как после этапа фильтрации по типам контента (`filteredItems`) список кандидатов становится пустым.

## Локализация

### Файлы

- `src/app/api/recommendations/random/route.ts` (основной API endpoint)
- `src/app/recommendations/RecommendationsClient.tsx` (клиентский компонент)
- `src/app/recommendations/RecommendationCard.tsx` (компонент карточки)

### Ключевые функции и участки кода

1. **`route.ts`**:
   - Функции `isAnime()` и `isCartoon()` (строки 136-160) используют поле `genre_ids` для проверки наличия жанра анимации (id=16):
     ```typescript
     const isAnimation = (data.genre_ids as number[] | undefined)?.includes(16) ?? false;
     ```
     Однако, передаваемый в них объект `detailsObj` имеет тип `MovieDetails` из `@/lib/tmdb`, который содержит поле `genres` (массив `{id, name}`), но НЕ содержит `genre_ids`. Поэтому `data.genre_ids` всегда `undefined`, `isAnimation` всегда `false`, и обе функции возвращают `false` для любых данных.
   - Логика фильтрации (строки 563-603) использует `details.isAnime` и `details.isCartoon`, которые из-за вышесказанного всегда `false`. При фильтрации по `anime` или `cartoon` ни один элемент не проходит.
   - Определение `displayMediaType` (строка 861) не учитывает `isCartoon`, возвращая `'movie'` или `'tv'` для мультфильмов:
     ```typescript
     const displayMediaType = isAnimeResult ? 'anime' : (selected.mediaType === 'movie' ? 'movie' : 'tv');
     ```

2. **`RecommendationsClient.tsx`**:
   - Тип `MovieData.media_type` (строка 22) не включает `'cartoon'`.
   - Вычисляется только `isAnime` (строки 452-454), отсутствует `isCartoon`.
   - При обработке ответа API нет проверки мультфильмов.

3. **`RecommendationCard.tsx`**:
   - Тип `MovieData.media_type` (строка 13) не включает `'cartoon'`.
   - Нет пропса `isCartoon`.
   - Логика подмены `media_type` для аниме (строка 148) может требоваться доработки для мультфильмов.

### Предполагаемая причина

Фундаментальное несоответствие структуры данных: функции `isAnime()` и `isCartoon()` ожидают поле `genre_ids`, но TMDB данные, предоставляемые через `MovieDetails`, хранят жанры в поле `genres` (массив объектов). Это приводит к постоянному `false` для проверки анимации, и как следствие, фильтрация по аниме и мультфильмам не находит кандидатов. Дополнительные проблемы с типами и отображением exacerbate issue.

### Context7 проверка

Не требовалась — проблема связана с внутренней логикой приложения, а не с внешними библиотеками.

## Spec для RED-теста

Тест должен воспроизвести сценарий, когда пользователь имеет в библиотеке аниме и/или мультфильмы и запрашивает рекомендацию с фильтром только этих типов.

**Действия:**
1. Создать пользователя с `watchList` содержащую:
   - Фильм (movie) с жанром анимации? (например, `genre_ids: [16, ...]`) и `original_language` не 'ja' (мультфильм).
   - Аниме (anime) с `genre_ids` включая 16 и `original_language = 'ja'`.
   - Обычный фильм/сериал без анимации.
2. Вызвать endpoint `GET /api/recommendations/random` с query параметрами:
   - `types=cartoon` (только мультфильмы)
   - `types=anime` (только аниме)
   - `types=anime,cartoon` (оба типа)
   - `types=movie,tv` (обычные фильмы/сериалы, должны игнорировать анимационные)
3. Проверить, что возвращаемый `movie` объект имеет `media_type` равный:
   - `'cartoon'` для мультфильмов
   - `'anime'` для аниме
   - `'movie'`/`'tv'` для обычных
4. Для фильтров `anime`/`cartoon` убедиться, что в ответе `success: true` и `movie` не `null`.
5. При фильтре `cartoon` убедиться, что обычные фильмы/сериалы не возвращаются.
6. Проверить, что в `stats` отображается корректное количество доступных кандидатов.

**Текущее падение:**
- При `types=anime` или `types=cartoon` `candidates.length === 0`, возвращается `success: false` с сообщением "Нет доступных рекомендаций...".

## Acceptance критерии

- [ ] **API фильтрация**: `isAnime()` и `isCartoon()` корректно определяют типы на основе `genres` массива TMDB данных (жанр 16) и `original_language`.
- [ ] **displayMediaType**: возвращает `'cartoon'` для мультфильмов, `'anime'` для аниме, в остальных случаях `'movie'`/`'tv'`.
- [ ] **Client types**: `MovieData.media_type` включает `'cartoon'` во всех клиентских файлах (`RecommendationsClient.tsx`, `RecommendationCard.tsx`).
- [ ] **Client state**: `RecommendationsClient` вычисляет `isCartoon` на основе `media_type === 'cartoon'` (или по тем же правилам, что и сервер) и передает в `RecommendationCard`.
- [ ] **RecommendationCard**: принимает `isCartoon` пропс и корректно отображает мультфильмы (badge "Мульт").
- [ ] **Regression**: существующие тесты проходят, новые тесты покрывают сценарии фильтрации аниме/мульт.
- [ ] **Manual testing**: пользователь может получить рекомендации, выбрав только "Аниме" или только "Мульт" или их комбинацию.
